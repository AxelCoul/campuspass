Connexion :

API backend (à prévoir si pas existant) :

6. DÉTAIL PAR ÉCRAN
--------------------------------------------------------------------------------

--- 1) Splash (/splash)
  - Logo PASS CAMPUS
  - Indicateur de chargement
  - Si token valide → Dashboard, sinon → Login

--- 2) Login (/login)
  - Champs : email, mot de passe
  - Bouton : Connexion
  - Lien : Mot de passe oublié
  - API : POST /api/auth/merchant/login (ou /api/auth/login + récup merchantId)

--- 3) Dashboard (/dashboard)
  - Cartes : Coupons utilisés aujourd’hui, Revenu du jour, Offres actives, Visites
  - Graphique : utilisation coupons par jour (optionnel)
  - API : à définir (ex. GET /api/merchant/dashboard ou agrégation côté app)

--- 4) Scanner (/scanner)
  - Ouverture caméra / scan QR code
  - Récupération du code coupon (ex. string dans le QR)
  - Appel API : POST /api/coupons/validate?merchantId=<id>  body: { "couponCode": "..." }
  - Selon réponse : navigation vers Coupon Validation (succès) ou message d’erreur
    (Coupon expiré, déjà utilisé, invalide)

--- 5) Coupon Validation (/coupon-validation)
  - Affichage : Nom étudiant, Offre, Réduction, Montant final
  - Bouton : Confirmer réduction (peut fermer l’écran et revenir au scanner)
  - Données : passées en paramètre depuis le résultat de validate

--- 6) Liste offres (/offers)
  - Cards : Image, Titre, Réduction, Date expiration, Statut
  - Actions : Modifier, Désactiver, Voir stats
  - API : GET /api/offers?merchantId=<id>

--- 7) Créer offre (/offers/create)
  - Formulaire : Titre, Description, Image, Réduction %, Prix initial,
    Date début, Date fin, Nombre max coupons
  - Bouton : Publier offre
  - API : POST /api/offers  (body selon OfferRequest backend)

--- 8) Modifier offre (/offers/edit/:id)
  - Même formulaire que création, pré-rempli
  - API : PUT /api/offers/:id

--- 9) Transactions (/transactions)
  - Liste : ID, Étudiant, Offre, Montant, Réduction, Date
  - Filtres : date, offre
  - API : GET /api/transactions (backend à exposer pour merchant par merchantId)

--- 10) Statistiques (/analytics)
  - Graphiques : coupons utilisés, revenus, offres populaires
  - API : GET /api/merchant/analytics ou agrégation

--- 11) Publicités (/ads)
  - Formulaire : Image, Position, Durée, Budget
  - API : à définir (création de campagnes pub)

--- 12) Notifications (/notifications)
  - Liste des messages : nouveaux étudiants, offres expirées, stats
  - API : GET /api/notifications (filtré par merchant si besoin)

--- 13) Profil commerce (/profile)
  - Infos : Nom commerce, Adresse, Téléphone, Email, Logo
  - Actions : Modifier profil, Changer mot de passe
  - API : GET /api/merchants/:id, PUT /api/merchants/:id

--- 14) Personnel (/staff)
  - Liste : Nom, Rôle, Statut
  - Actions : Ajouter employé, Modifier rôle, Supprimer
  - API : à définir (staff par merchant)

--- 15) Paramètres (/settings)
  - Options : Notifications, Langue, Sécurité

--- 16) Support (/support)
  - Contacter support, FAQ, Ouvrir ticket

8. INTERACTIONS BACKEND (APIs)
--------------------------------------------------------------------------------

Authentification
  - POST /api/auth/merchant/login  { "email", "password" }
    → à créer si pas existant (sinon login + GET merchants?ownerId=)
  - Réponse attendue : { "token", "id", "email", "firstName", "lastName", "role", "merchantId" }

Merchant
  - GET  /api/merchants?ownerId=<userId>   → liste (1 commerce pour 1 owner)
  - GET  /api/merchants/:id
  - PUT  /api/merchants/:id   (modifier profil)

Offres
  - GET  /api/offers?merchantId=<id>
  - GET  /api/offers/:id
  - POST /api/offers
  - PUT  /api/offers/:id
  - DELETE /api/offers/:id

Coupons (validation par le commerce)
  - POST /api/coupons/validate?merchantId=<id>   body: { "couponCode": "..." }
  - Réponse : CouponResponse (détails coupon, offre, etc.)

Transactions
  - GET /api/transactions  (pour le merchant connecté)
  - À exposer côté backend : GET /api/merchants/:id/transactions ou
    GET /api/transactions?merchantId=<id> avec auth MERCHANT

Dashboard / Analytics
  - À définir : GET /api/merchant/dashboard, GET /api/merchant/analytics
  - Ou calcul côté app à partir des offres, coupons, transactions

Notifications
  - GET /api/notifications (si existant, filtré par utilisateur ou merchant)
