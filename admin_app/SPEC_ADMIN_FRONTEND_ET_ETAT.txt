7. DÉTAIL PAR ÉCRAN (OBJECTIF, UI, ACTIONS)
--------------------------------------------------------------------------------

1) Login Admin (/admin/login)
   - Logo PASS CAMPUS, email, mot de passe, bouton Login, lien reset password.

2) Dashboard (/admin/dashboard)
   - Widgets : Total étudiants, commerces, offres actives, coupons utilisés aujourd'hui, revenus.
   - Graphiques : utilisation coupons, croissance utilisateurs, commerces populaires.

3) Liste utilisateurs (/admin/users)
   - Table : ID, Nom, Email, Role, Statut, Date inscription.
   - Actions : voir profil, suspendre, supprimer, réactiver.

4) Profil utilisateur (/admin/users/:id)
   - Infos : Nom, Email, Téléphone, Ville, Historique coupons, Transactions.
   - Actions : suspendre, réactiver.

5) Liste étudiants (/admin/students)
   - Colonnes : Nom, Université, Ville, Points, Coupons utilisés, Statut.
   - Filtres : ville, université, statut.
   - Actions : valider carte étudiant, suspendre compte.

6) Liste commerces (/admin/merchants)
   - Table : Nom commerce, Propriétaire, Ville, Catégorie, Statut, Date inscription.
   - Actions : approuver, refuser, voir détails, suspendre.

7) Détail commerce (/admin/merchants/:id)
   - Sections : Informations, Images, Offres, Transactions, Avis.
   - Actions : approuver, suspendre, supprimer.

8) Liste offres (/admin/offers)
   - Table : Image, Titre, Commerce, Réduction, Date début/fin, Statut.
   - Actions : modifier, supprimer, désactiver.

9) Créer offre (/admin/offers/create)
   - Formulaire : Titre, Description, Commerce, Catégorie, Image, Réduction %, Prix initial, Date début/fin.

10) Modifier offre (/admin/offers/edit/:id)
    - Même formulaire que création.

11) Liste coupons (/admin/coupons)
    - Table : Coupon ID, Étudiant, Offre, Statut (généré/utilisé/expiré), Date génération, Date utilisation.

12) Transactions (/admin/transactions)
    - Table : Transaction ID, Commerce, Étudiant, Montant initial, Réduction, Montant final, Date.
    - Filtres : date, commerce, statut.

13) Paiements (/admin/payments)
    - Table : Payment ID, Transaction, Montant, Méthode (mobile money, card, cash), Statut, Date.

14) Publicités (/admin/advertisements)
    - Cards : Image, Titre, Position, Statut, Date début/fin.
    - Actions : modifier, supprimer, activer. Créer publicité, upload image, position, planifier.

15) Catégories (/admin/categories)
    - Liste : Restaurant, Mode, Electronique, Supermarché, Transport.
    - Actions : créer, modifier, supprimer.

16) Avis (/admin/reviews)
    - Table : Utilisateur, Commerce, Note, Commentaire, Date.
    - Actions : supprimer, masquer, signaler.

17) Notifications (/admin/notifications)
    - Formulaire : Titre, Message, Audience (tous étudiants, ville, commerce), Date envoi.

18) Analytics (/admin/analytics)
    - Graphiques : croissance utilisateurs, utilisation coupons, revenus, offres populaires.

19) Gestion admins (/admin/admins)
    - Table : Nom, Email, Role, Statut.
    - Actions : créer admin, modifier rôle, désactiver.

20) Paramètres (/admin/settings)
    - Commission plateforme, durée coupons, configuration email, notifications.

21) Logs système (/admin/logs)
    - Optionnel : consultation des logs.

8. ARCHITECTURE ANGULAR
--------------------------------------------------------------------------------
src/app

  - layouts       (admin-layout avec sidebar + header)
  - services      (API : users, students, merchants, offers, coupons, etc.)

  - modules       (si besoin)
  - pages         (une page par écran)
  - shared        (composants partagés)

Nombre indicatif : Pages principales ~20, Composants UI ~40, Services API ~15, Guards ~3.

--------------------------------------------------------------------------------
✅ DÉJÀ FAIT (ADMIN_APP)
--------------------------------------------------------------------------------

• ROUTES ET LAYOUT
  - Routes : login, dashboard, users, students, merchants, merchants/:id,
    offers, offers/create, offers/edit/:id, coupons, transactions, payments,
    advertisements, categories, reviews, notifications, analytics, admins, settings.
  - Layout : sidebar fixe + header avec déconnexion et email utilisateur.
  - Pas de route /admin/logs pour l’instant.

• AUTHENTIFICATION
  - Login avec email/mot de passe, appel POST auth/admin/login.
  - JWT stocké (admin_token, admin_user), AuthService, AuthGuard, AuthInterceptor.
  - Lien "Mot de passe oublié ?" présent en UI (lien #, non fonctionnel).

• DESIGN GLOBAL
  - Variables CSS : --primary #2D6CDF, --secondary #00C48C, --background #F4F6F9,
    --card #FFFFFF, --danger #FF4D4F, --warning #FFC107, --text, sidebar, header.
  - Cartes, tableaux, boutons, badges, formulaires de base.

• PAGES IMPLÉMENTÉES (avec données / UI réelles)
  - Login : formulaire + appel API.
  - Dashboard : widgets (étudiants, commerces, offres actives, coupons aujourd’hui,
    transactions, revenus) + message de bienvenue (pas encore de vrais graphiques).
  - Users : liste en tableau (ID, Nom, Email, Rôle, Statut, Inscription), chargement.
  - Merchants : liste + colonne Actions (Voir détail), statuts (APPROVED/PENDING/…).
  - Merchant Detail : page existante (à vérifier contenu).
  - Offers : liste + bouton "Créer une offre" + lien Modifier vers edit/:id.
  - Offer Form : create/edit avec formulaire et redirection après succès.
  - Coupons, Transactions, Payments, Advertisements, Categories, Reviews,
    Notifications, Analytics, Settings : pages présentes (à détailler selon le code).

• SERVICES API
  - users.service, dashboard.service, merchants.service, offers.service,
    coupons.service, transactions.service, payments.service, categories.service.
  - Appels vers /admin/users, /admin/dashboard/stats, /admin/merchants, etc.

• GUARDS
  - authGuard : redirection vers /admin/login si non authentifié.
  - roleGuard : vérification des rôles (hasRole), redirection vers dashboard si non autorisé.
  - Les routes admin n’utilisent pas encore le roleGuard par route (pas de restriction par SUPER_ADMIN / ADMIN_OPERATIONS / ADMIN_SUPPORT).
